1. 🧠 Query Understanding (antes do retrieval)

Em RAG avançado, a query não vai direta para search.

Ela passa por:

✔ Query rewriting
melhora a pergunta
resolve ambiguidades
"Idyn o que é?"
→ "Definição de corrente dinâmica estipulada Idyn em engenharia elétrica"
✔ Query decomposition

Divide perguntas complexas:

"Como funciona EU Kids Online e quais os resultados?"

→ Q1: O que é EU Kids Online?
→ Q2: Quais resultados do relatório?
✔ Intent classification

Define estratégia:

factual → BM25 + dense
conceptual → dense + HyDE
multi-hop → decomposition


🔀 Multi-stage retrieval real

Em vez de 2 retrievers (BM25 + dense), tens:

Query → (BM25 + Dense + Graph + Web + Memory)
                     ↓
                  Fusion
                     ↓

Tipos de retrieval:

BM25 (lexical)
Dense (semantic)
Multi-vector (ColBERT-style)
Graph retrieval (KG / knowledge graph)
Web retrieval (live)
Structured DB retrieval


3. 🧭 Context expansion (muito importante)

Em RAG avançado, não recuperas só chunks:

✔ Parent-child retrieval
recuperas chunk pequeno
sobes para secção inteira
chunk → parágrafo → secção → documento
✔ Window expansion

Se apanhas um chunk relevante:

adicionas contexto anterior e seguinte
✔ Document stitching

Reconstrói partes completas do raciocínio


4. 🧠 Reranking forte (nível produção)

Tu já tens Cross Encoder 👍

Mas versões avançadas usam:

Cross encoder + calibration
LLM reranking (sim, LLM escolhe top-K)
learning-to-rank models
diversity-aware ranking (MMR + semantic coverage)
