🧠 1. Parsing “clássico” (baseline ingénuo)
📌 Estratégia
regex
split por linhas
heurísticas simples
PDF → texto direto
Exemplos:
PyMuPDF
pdfplumber
BeautifulSoup (HTML)

PyMuPDF
pdfplumber

👍 Vantagens
rápido
barato
previsível
👎 Problemas
perde estrutura semântica
não entende contexto
chunking fica “cego”
🧱 2. Layout-aware parsing (estrutura visual)
📌 Ideia

Aqui já não olhas só para texto — olhas para posição no documento.

Técnicas:
detetar headings por tamanho de fonte
blocos por coordenadas (x/y)
leitura por ordem visual
Ferramentas:
PyMuPDF (com blocks)
pdfminer layout mode
layout parsing heuristics
O que melhora:

✔ hierarquia
✔ títulos vs texto
✔ tabelas básicas

👁️ 3. Vision-based parsing (Document AI)

Aqui começa o nível “IA Engineer sério”.

📌 Ideia

Transformar o documento em imagem e usar modelos de visão.

Modelos típicos:
LayoutLM / LayoutLMv3
Donut
Pix2Struct

LayoutLM
Document AI

O que faz:
deteta blocos semânticos
entende tabelas complexas
reconstrói ordem de leitura
extrai entidades estruturadas
👍 Muito forte em:
PDFs escaneados
relatórios complexos
formulários
🧠 4. LLM-based parsing (estruturação semântica)

Aqui entra o mundo moderno.

📌 Ideia

Usar um LLM para “entender e reorganizar” o documento.

Exemplo de pipeline:
chunk bruto
mandar para LLM
pedir output estruturado

Exemplo prompt:

Transforma este texto em JSON com:
- título
- secções
- entidades
- resumo
Vantagens:

✔ sem regras fixas
✔ adapta-se a qualquer domínio
✔ ótimo para dados ruidosos

Problema:
custo
inconsistência
não determinístico
🧩 5. Hierarchical parsing (nível RAG moderno)

Isto é o que sistemas bons usam.

📌 Ideia

Manter estrutura em árvore:

Documento
 ├── Secção
 │     ├── Subsecção
 │     ├── Parágrafo
Estratégia:
chunk por headings
sub-chunks por tokens
metadata rica
Usado em:
RAG sério
search engines
knowledge bases
🔍 6. Semantic chunking (embedding-driven)

Aqui o parsing já não é só estrutura — é significado.

📌 Ideia

Separar chunks quando o significado muda, não quando o texto muda.

Como funciona:
sliding window
embeddings
cosine similarity
break when similarity drops
Exemplo:

mesmo parágrafo → 2 tópicos diferentes → split automático

👍 Muito bom para:
artigos longos
docs técnicos
logs
🧠 7. Agentic parsing (nível topo)

Aqui já estás a usar agentes.

📌 Ideia

Um LLM decide como fazer parsing.

Exemplo:

“este PDF parece técnico”
“vou extrair tabelas separadamente”
“vou criar schema específico”
Ferramentas:
tool calling
multi-step reasoning
planners
